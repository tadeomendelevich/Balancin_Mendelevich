Guía para leer el log de datos WiFi en Qt (C++)
================================================

Este archivo describe cómo recibir y decodificar los datos de telemetría enviados por el robot a través de WiFi mediante el protocolo UDP.

1. Estructura de Datos (Binary)
-------------------------------
El robot envía una estructura binaria compacta (`WifiLogData_t`) encapsulada en el protocolo UNER.

**Estructura C en el microcontrolador (packed):**

```c
typedef struct __attribute__((packed)) {
    uint32_t t_ms;        // Tiempo en ms desde el inicio
    float    roll_filt;   // Ángulo de inclinación filtrado (grados)
    float    output;      // Salida del PID (antes de ganancia motor)
    float    p_term;      // Término Proporcional
    float    i_term;      // Término Integral
    float    d_term;      // Término Derivativo
    int16_t  mR;          // Velocidad Motor Derecho (-100 a 100)
    int16_t  mL;          // Velocidad Motor Izquierdo (-100 a 100)
} WifiLogData_t;
```

**Tamaño del Payload:** 28 bytes.

2. Protocolo de Trama UNER
--------------------------
La trama UDP completa tiene el siguiente formato:

| Byte | Valor / Significado          |
|------|------------------------------|
| 0    | 'U'                          |
| 1    | 'N'                          |
| 2    | 'E'                          |
| 3    | 'R'                          |
| 4    | Longitud (Payload + 1) = 29  |
| 5    | ':'                          |
| 6    | CMD = 0xBB (CMD_WIFI_LOG_DATA)|
| 7..34| Payload (WifiLogData_t)      |
| 35   | Checksum (XOR)               |

3. Configuración en Qt
----------------------
Para recibir estos datos, necesitas un `QUdpSocket` escuchando en el puerto local configurado (por defecto 30000).

**Pasos:**
1. Agregar `network` al archivo `.pro`: `QT += network`
2. Crear un socket UDP y enlazarlo al puerto.
3. Procesar los datagramas entrantes.

**Clase Completa `UdpTelemetryReceiver` (Ejemplo):**

```cpp
// UdpTelemetryReceiver.h
#ifndef UDPTELEMETRYRECEIVER_H
#define UDPTELEMETRYRECEIVER_H

#include <QObject>
#include <QUdpSocket>
#include <QNetworkDatagram>

// Estructura equivalente en Qt (debe coincidir con la del microcontrolador)
#pragma pack(push, 1)
struct WifiLogData {
    quint32 t_ms;
    float   roll_filt;
    float   output;
    float   p_term;
    float   i_term;
    float   d_term;
    qint16  mR;
    qint16  mL;
};
#pragma pack(pop)

class UdpTelemetryReceiver : public QObject
{
    Q_OBJECT
public:
    explicit UdpTelemetryReceiver(QObject *parent = nullptr);
    void startListening(quint16 port = 30000);
    void sendStartLogCommand(const QString &robotIp, quint16 robotPort = 30010);

signals:
    void newDataReceived(WifiLogData data);

private slots:
    void processPendingDatagrams();

private:
    QUdpSocket *udpSocket;
};

#endif // UDPTELEMETRYRECEIVER_H
```

```cpp
// UdpTelemetryReceiver.cpp
#include "UdpTelemetryReceiver.h"
#include <QDebug>

UdpTelemetryReceiver::UdpTelemetryReceiver(QObject *parent) : QObject(parent)
{
    udpSocket = new QUdpSocket(this);
    connect(udpSocket, &QUdpSocket::readyRead, this, &UdpTelemetryReceiver::processPendingDatagrams);
}

void UdpTelemetryReceiver::startListening(quint16 port)
{
    if (udpSocket->bind(QHostAddress::Any, port)) {
        qDebug() << "Escuchando telemetría UDP en el puerto" << port;
    } else {
        qDebug() << "Error al abrir puerto UDP:" << udpSocket->errorString();
    }
}

void UdpTelemetryReceiver::sendStartLogCommand(const QString &robotIp, quint16 robotPort)
{
    // Trama para activar el log: UNER + len + : + CMD (0xBA) + Checksum
    // CMD_ACTIVATE_WIFI_LOG = 0xBA
    // Len = 1 (CMD) + 1 (Payload vacío, opcional) = 2? No, len es payload+1. Payload es 0. Len=1.
    // Revisando UNER.c: len es length (payload) + 1. Si payload es 0, len=1.

    QByteArray packet;
    packet.append("UNER");
    packet.append((char)0x01); // Len = 1 (Solo CMD)
    packet.append(':');
    packet.append((char)0xBA); // CMD_ACTIVATE_WIFI_LOG

    // Checksum: 'U'^'N'^'E'^'R'^len^':'^cmd
    char chk = 'U' ^ 'N' ^ 'E' ^ 'R' ^ 0x01 ^ ':' ^ 0xBA;
    packet.append(chk);

    udpSocket->writeDatagram(packet, QHostAddress(robotIp), robotPort);
    qDebug() << "Comando de inicio enviado a" << robotIp;
}

void UdpTelemetryReceiver::processPendingDatagrams()
{
    while (udpSocket->hasPendingDatagrams()) {
        QNetworkDatagram datagram = udpSocket->receiveDatagram();
        QByteArray data = datagram.data();

        // Validar cabecera básica "UNER" y tamaño mínimo (Header 7 + Payload 28 + Chk 1 = 36)
        if (data.size() < 36 || !data.startsWith("UNER")) continue;

        // Verificar comando (Byte 6) -> 0xBB (CMD_WIFI_LOG_DATA)
        if ((quint8)data[6] != 0xBB) continue;

        // Extraer payload (Bytes 7 a 34)
        WifiLogData log;
        // Copiar de forma segura
        memcpy(&log, data.constData() + 7, sizeof(WifiLogData));

        // Emitir señal con los datos decodificados
        emit newDataReceived(log);
    }
}
```

4. Integración en MainWindow
---------------------------
En tu `MainWindow.cpp`:

```cpp
// 1. Instanciar
receiver = new UdpTelemetryReceiver(this);

// 2. Conectar señal a tu slot de actualización de GUI/Gráficos
connect(receiver, &UdpTelemetryReceiver::newDataReceived, this, [](WifiLogData data){
    qDebug() << "Roll:" << data.roll_filt << " PID:" << data.output;
    // ui->plot->addData(data.t_ms, data.roll_filt);
});

// 3. Iniciar escucha
receiver->startListening(30000);

// 4. En algún botón "Conectar/Iniciar Log":
receiver->sendStartLogCommand("192.168.1.XX", 30010);
```

Recuerda reemplazar `"192.168.1.XX"` con la IP real de tu robot (la que configuraste en `main.c`).
